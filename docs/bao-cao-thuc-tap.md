## 1. LỜI MỞ ĐẦU

Trong quá trình học tập tại trường Đại học, việc thực tập tại doanh nghiệp là một phần quan trọng giúp sinh viên tiếp cận với môi trường làm việc thực tế, áp dụng kiến thức đã học vào thực tiễn và phát triển kỹ năng nghề nghiệp.

Với mong muốn được trải nghiệm và học hỏi kiến thức thực tế trong lĩnh vực phát triển phần mềm, em đã có cơ hội được thực tập tại [Tên công ty] trong [X] tuần. Trong thời gian này, em được tham gia vào dự án phát triển hệ thống Cloody - một ứng dụng quản lý và chia sẻ tệp tin trực tuyến.

Báo cáo này trình bày tổng quan về quá trình thực tập, các kiến thức và kỹ năng em đã học được, cũng như những khó khăn đã gặp phải và cách giải quyết trong suốt thời gian thực tập tại công ty.

- Tải lên, lưu trữ và quản lý tệp/thư mục

- Tìm kiếm và lọc dữ liệu nhanh chóng

**Ý nghĩa của dự án:**

##### **A. Module quản lý người dùng (User Management)**

**Chức năng đăng ký và đăng nhập:**
- Tạo form đăng ký với validation (email, password, confirm password)
- Xử lý logic đăng ký: hash password, lưu vào database
- Tạo form đăng nhập và xác thực người dùng
- Sử dụng Laravel Authentication để quản lý session
- Implement chức năng "Remember Me" và "Forgot Password"

**Code ví dụ - Registration Controller:**
```php
public function register(Request $request)
{
    $validated = $request->validate([
        'name' => 'required|string|max:255',
        'email' => 'required|string|email|max:255|unique:users',
        'password' => 'required|string|min:8|confirmed',
    ]);

    $user = User::create([
        'name' => $validated['name'],
        'email' => $validated['email'],
        'password' => Hash::make($validated['password']),
    ]);

    Auth::login($user);
    
    return redirect()->route('dashboard');
}
```

**Chức năng quản lý hồ sơ cá nhân:**
- Xem và cập nhật thông tin cá nhân (tên, email, địa chỉ, ảnh đại diện)
- Upload và xử lý ảnh đại diện
- Đổi mật khẩu với xác thực mật khẩu cũ
- Validation dữ liệu đầu vào

**Những gì đã học:**
- Cách sử dụng Laravel Validation
- Hash và bảo mật mật khẩu với bcrypt
- Xử lý upload file và lưu trữ
- Session và Cookie management
- Middleware authentication

##### **B. Module quản lý thư mục (Folder Management)**

**Chức năng tạo và quản lý thư mục:**
- Tạo thư mục mới với tên và thư mục cha (parent folder)
- Hiển thị danh sách thư mục theo cấu trúc cây
- Đổi tên thư mục
- Di chuyển thư mục
- Xóa mềm thư mục (soft delete với trường `trashed_at`)

**Code ví dụ - Create Folder:**
```php
public function store(Request $request)
{
    $validated = $request->validate([
        'name' => 'required|string|max:255',
        'parent_id' => 'nullable|exists:folders,id',
        'privacy_setting' => 'required|in:private,shared,public'
    ]);

    $folder = Folder::create([
        'name' => $validated['name'],
        'parent_id' => $validated['parent_id'],
        'user_id' => Auth::id(),
        'privacy_setting' => $validated['privacy_setting'],
    ]);

    return redirect()->back()->with('success', 'Folder created successfully');
}
```

**Chức năng nâng cao:**
- Đánh dấu thư mục yêu thích (is_favorite)
- Cài đặt quyền riêng tư cho thư mục (private/shared/public)
- Breadcrumb navigation để điều hướng giữa các thư mục
- Khôi phục thư mục từ thùng rác

**Những gì đã học:**
- Eloquent Relationships (hasMany, belongsTo)
- Soft Deletes trong Laravel
- Recursive queries cho cấu trúc cây
- Authorization và Policy

##### **C. Module quản lý tệp tin (File Management)**

**Chức năng upload tệp:**
- Form upload file với progress bar
- Validation: loại file, kích thước tối đa
- Lưu file vào storage với tên unique
- Lưu metadata vào database (tên gốc, kích thước, loại file, đường dẫn)
- Cấu hình upload limit trong PHP và Laravel

**Code ví dụ - Upload File:**
```php
public function upload(Request $request)
{
    $request->validate([
        'file' => 'required|file|max:51200', // 50MB
        'folder_id' => 'required|exists:folders,id'
    ]);

    $file = $request->file('file');
    $filename = time() . '_' . Str::random(10) . '.' . $file->getClientOriginalExtension();
    $path = $file->storeAs('uploads', $filename, 'public');

    File::create([
        'name' => $file->getClientOriginalName(),
        'filename' => $filename,
        'path' => $path,
        'size' => $file->getSize(),
        'mime_type' => $file->getMimeType(),
        'folder_id' => $request->folder_id,
        'user_id' => Auth::id(),
    ]);

    return response()->json(['success' => true]);
}
```

**Chức năng quản lý tệp:**
- Hiển thị danh sách file theo folder
- Download file
- Xem preview (cho image, PDF)
- Đổi tên file
- Di chuyển file giữa các folder
- Xóa mềm và khôi phục file

**Cấu hình upload limit:**
- Chỉnh sửa `php.ini`: `upload_max_filesize`, `post_max_size`
- Cấu hình Nginx/Apache
- Validation trong Laravel

**Những gì đã học:**
- Laravel Storage và Filesystem
- File handling và validation
- Symbolic link (storage:link)
- MIME type detection
- Response download và streaming

##### **D. Module chia sẻ (Sharing)**

**Chức năng chia sẻ tệp/thư mục:**
- Chia sẻ với người dùng cụ thể qua email
- Cài đặt quyền: view only, download
- Tạo link chia sẻ công khai
- Quản lý danh sách người được chia sẻ
- Thu hồi quyền chia sẻ

**Code ví dụ - Share Folder:**
```php
public function shareFolder(Request $request, $folderId)
{
    $validated = $request->validate([
        'user_email' => 'required|email|exists:users,email',
        'permission' => 'required|in:view,download,edit'
    ]);

    $folder = Folder::findOrFail($folderId);
    $this->authorize('share', $folder);

    $shareUser = User::where('email', $validated['user_email'])->first();

    FolderShare::create([
        'folder_id' => $folderId,
        'user_id' => $shareUser->id,
        'shared_by' => Auth::id(),
        'permission' => $validated['permission'],
    ]);

    // Send email notification
    Mail::to($shareUser)->send(new FolderSharedMail($folder));

    return redirect()->back()->with('success', 'Folder shared successfully');
}
```

**Những gì đã học:**
- Many-to-Many relationships
- Pivot tables
- Email notification với Laravel Mail
- Authorization policies
- Share link generation và security

##### **F. Module tìm kiếm và lọc**

**Chức năng:**
- Tìm kiếm file/folder theo tên
- Lọc theo loại file (document, image, video...)
- Lọc theo danh mục
- Lọc theo ngày tạo
- Lọc file yêu thích

**Code ví dụ - Search:**
```php
public function search(Request $request)
{
    $query = File::query()->where('user_id', Auth::id());

    if ($request->has('keyword')) {
        $query->where('name', 'like', '%' . $request->keyword . '%');
    }

    if ($request->has('category')) {
        $query->whereHas('categories', function($q) use ($request) {
            $q->where('id', $request->category);
        });
    }

    if ($request->has('type')) {
        $query->where('mime_type', 'like', $request->type . '%');
    }

    $files = $query->orderBy('created_at', 'desc')->paginate(20);

    return view('files.search', compact('files'));
}
```

**Những gì đã học:**
- Query builder advanced
- Eloquent relationships trong queries
- Pagination
- Search optimization

## 7. KHÓ KHĂN VÀ CÁCH GIẢI QUYẾT

### 7.1. Khó khăn về kỹ thuật

#### **Vấn đề 1: Chưa quen với Laravel và MVC**

**Khó khăn:**
- Lần đầu tiên làm việc với Laravel framework
- Chưa hiểu rõ cấu trúc MVC và flow của request
- Bối rối với nhiều folders và files

**Cách giải quyết:**
- Xem Laravel documentation từng phần một
- Xem video tutorials trên YouTube và Laracasts
- Đọc và phân tích code có sẵn trong dự án
- Hỏi mentor và senior developer
- Làm các tutorial project nhỏ để thực hành

**Bài học:**
- Học framework mới cần kiên trì và thực hành nhiều
- Documentation là nguồn tài liệu quan trọng nhất
- Học từ code có sẵn rất hiệu quả

#### **Vấn đề 2: Xử lý upload file lớn**

**Khó khăn:**
- Upload file lớn bị timeout
- Không hiểu cách cấu hình upload limit
- File upload thành công nhưng không lưu được vào database

**Cách giải quyết:**
- Research về PHP upload configuration: `upload_max_filesize`, `post_max_size`, `max_execution_time`
- Chỉnh sửa file `php.ini` và restart server
- Cấu hình validation trong Laravel
- Debug bằng cách log và kiểm tra từng bước
- Hỏi senior và được hướng dẫn cách handle errors

**Bài học:**
- Server configuration cũng quan trọng như code
- Cần kiểm tra logs để tìm lỗi
- Chia nhỏ vấn đề để debug dễ dàng hơn

#### **Vấn đề 3: N+1 Query Problem**

**Khó khăn:**
- Trang load chậm khi hiển thị nhiều files
- Không biết vấn đề nằm ở đâu
- Chưa hiểu về query optimization

**Cách giải quyết:**
- Cài đặt Laravel Debugbar để xem số lượng queries
- Phát hiện có hàng trăm queries cho một trang
- Học về Eager Loading: `with()`, `load()`
- Apply eager loading: `File::with('folder', 'user')->get()`
- Kết quả: giảm từ 200+ queries xuống còn 5 queries

**Bài học:**
- Performance optimization rất quan trọng
- Cần monitor và measure trước khi optimize
- Eloquent relationships cần được load đúng cách

#### **Vấn đề 4: Git conflicts**

**Khó khăn:**
- Lần đầu gặp merge conflict khi pull code
- Không biết cách resolve conflict
- Sợ làm mất code của người khác

**Cách giải quyết:**
- Học về Git workflow: branch, merge, rebase
- Được senior hướng dẫn cách resolve conflict
- Sử dụng Git GUI tool để visualize conflicts
- Thực hành nhiều lần và cẩn thận khi merge

**Bài học:**
- Git là kỹ năng bắt buộc phải thành thạo
- Commit code thường xuyên và pull code mỗi ngày
- Giao tiếp với team để tránh conflict về logic

### 7.2. Khó khăn về giao tiếp và làm việc nhóm

#### **Vấn đề 1: Ngại hỏi khi gặp khó khăn**

**Khó khăn:**
- Sợ hỏi nhiều sẽ làm phiền mọi người
- Tự mày mò quá lâu không giải quyết được
- Ảnh hưởng đến tiến độ công việc

**Cách giải quyết:**
- Mentor động viên và khuyến khích hỏi
- Học cách đặt câu hỏi đúng: cụ thể, có context, đã thử gì
- Hỏi sau khi đã research và không tìm ra giải pháp
- Nhận ra rằng hỏi là cách học nhanh nhất

**Bài học:**
- Hỏi đúng lúc, đúng người giúp tiết kiệm thời gian
- Mọi người đều sẵn lòng giúp đỡ
- Đặt câu hỏi tốt cũng là một kỹ năng

#### **Vấn đề 2: Ước lượng thời gian không chính xác**

**Khó khăn:**
- Estimate task hoàn thành trong 1 ngày nhưng thực tế mất 3 ngày
- Gây delay cho sprint
- Áp lực khi không hoàn thành đúng hạn

**Cách giải quyết:**
- Học cách break down task thành các subtasks nhỏ
- Estimate từng subtask cụ thể hơn
- Thêm buffer time cho những việc không dự đoán được
- Báo cáo sớm nếu thấy có risk delay

**Bài học:**
- Ước lượng là kỹ năng cần thời gian để học
- Nên ước lượng thận trọng hơn khi mới bắt đầu
- Communication sớm giúp team có kế hoạch backup

### 7.3. Khó khăn về thời gian

**Vấn đề:** Cân bằng giữa học tập, thực tập và các môn học khác ở trường

**Cách giải quyết:**
- Lập kế hoạch và prioritize công việc
- Focus 100% trong giờ làm việc
- Tận dụng thời gian tối đa để học thêm
- Xin phép được flexible về thời gian khi cần

- Kỹ năng mềm (soft skills) quan trọng không kém technical skills

   - Luôn học hỏi và cập nhật kiến thức mới

   - Học từ code của người khác

   - Communication là chìa khóa trong teamwork
   - Hỏi khi cần thiết, đừng tự struggle quá lâu
   - Commit code thường xuyên
   - Testing và code review rất quan trọng

3. **Về phát triển bản thân:**
   - Đặt mục tiêu rõ ràng cho sự nghiệp
   - Không ngừng học hỏi và cải thiện
   - Tham gia community và networking
   - Work-life balance cũng quan trọng

4. **Về nghề nghiệp:**
   - Lập trình viên cần cả technical và soft skills
   - Thị trường IT rất rộng mở với người có năng lực
   - Đam mê là động lực quan trọng để phát triển
   - Team spirit và collaboration rất được coi trọng

### 8.5. Kế hoạch phát triển sau thực tập

**Ngắn hạn (3-6 tháng):**

- Làm thêm các pet projects để practice
- Học sâu hơn về design patterns và architecture
- Cải thiện kỹ năng testing và debugging
- Học thêm về DevOps cơ bản (CI/CD, Docker)

**Trung hạn (6-12 tháng):**

- Tham gia các dự án open source
- Build portfolio và blog cá nhân
- Networking và tham gia tech communities

**Dài hạn (1-3 năm):**
- Phát triển lên Middle Developer
- Chuyên sâu vào một mảng (Backend, Frontend, Full-stack)
- Mentor cho junior developers
- Có khả năng lead small projects
- Không ngừng học hỏi và cập nhật công nghệ
